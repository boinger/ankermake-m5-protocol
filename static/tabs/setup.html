External Camera Setup
Use the main stream if the camera exposes RTSP, MJPEG, or another FFmpeg-readable feed.
Optional. If your camera provides a direct JPEG snapshot endpoint, use it here for faster previews and capture.
External camera settings are stored per printer, so each printer can use its own feed if needed.
Setup Notes

External camera support is meant for simple FFmpeg-friendly feeds.

  • RTSP example: rtsp://user:pass@camera-ip:554/stream1
  • MJPEG example: http://camera-ip/mjpeg
  • Snapshot example: http://camera-ip/snapshot.jpg

Use the Home page to switch between the printer camera and the external feed. Snapshot and timelapse follow that same source.

External preview updates as refreshed still frames rather than the low-latency PPPP stream used by the printer camera.

Windows Launcher
Enter the folder that contains ankerctl.py. The downloaded batch file will launch from this folder.
This launcher uses your path, changes into that folder, and runs python .\ankerctl.py webserver run.
How To Use It
  1. Confirm the Ankerctl folder path.
  2. Download the batch file.
  3. Place it wherever you want on your PC.
  4. Double-click it to start Ankerctl.

The batch file is generated specifically from the path you enter here, so you do not have to edit it afterward.

If you move the Ankerctl folder later, just come back here and download a fresh launcher.
